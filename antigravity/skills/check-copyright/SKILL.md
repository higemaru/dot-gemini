name: check-copyright
description: 著作物に関するチェックを行う。A)自分の原稿が他者の著作物に似すぎていないか、B)引用・参照の扱いが適切か。
---

## 他のチェックスキルとの使い分け
- /check-text      → 誤字・脱字・用語統一（機械的チェック）
- /check-style     → DNAとの乖離（文体・トーン・構造の質的チェック）
- /check-copyright → 著作物に関する確認（類似・引用）

## 実行手順

1. 対象ファイルを確認する
   - 指定があればそのファイル
   - なければ直近の編集ファイル

2. references/ または resources/ にある参照・引用リストを読む
   - ファイルが存在しない場合はその旨を記録してB項目をスキップ

3. 以下の2セクションでチェックし、レポートを出す

---

### A: 表現の類似チェック

- **フレーズの類似**: 特定の作品・楽曲・広告コピーなどに由来しそうな表現がないか
  - 例: 有名なキャッチコピーや歌詞に近い言い回し

- **構造・展開の類似**: 既存作品のストーリー構造やキャラクター設定に強く依存していないか
- **固有名詞・造語**: 他者の著作物に登場する固有名詞・造語をそのまま使っていないか

報告形式:
```
【A: 表現の類似チェック】
△ 「〜〜〜」 → （何に似ているか・なぜ気になるか）
問題なし → 「類似の懸念なし」
```

---

### B: 引用・参照の適切性チェック

- **出典の明記**: 引用元が明記されているか
- **引用の範囲**: 必要最小限にとどまっているか
- **references/ との整合**: 原稿中の引用が references/ に記録されているか
  - 原稿にあって references/ にない → 「要記録」として報告
  - references/ にあって原稿に使われていない → 報告不要

報告形式:
```
【B: 引用・参照チェック】
要確認 「〜〜〜」 → （問題の内容）
要記録 「〜〜〜」 → references/ への追加を推奨
問題なし → 「引用・参照の問題なし」
```

---

4. 全項目問題なければ「著作権チェック完了：問題なし」で締める

## 注意
- 法的判断はできない。「気になる箇所」の指摘にとどめる
- 「問題ない」と断言しない。あくまでチェックの補助
- 指摘が多い場合は優先度の高いものから最大10箇所まで

## 他のチェックスキルとの使い分け
- /check-text      → 誤字・脱字・用語統一（機械的チェック）
- /check-style     → DNAとの乖離（文体・トーン・構造の質的チェック）
- /check-copyright → 著作物に関する確認（類似・引用）

## 実行手順

1. CLAUDE.md を読んでジャンルを確認し、対応する DNA ファイルを読む
   - 小説: @~/.claude/rules/novel-dna.md
   - 技術書: @~/.claude/rules/technical-dna.md

2. 対象ファイルを確認する
   - 指定があればそのファイル
   - なければ直近の編集ファイル

3. DNA の各項目と原稿を照合してレポートを出す

   **小説の確認項目**
   - [ ] 感情語を直接使いすぎていないか
   - [ ] 括弧の使い分けが DNA のルールに従っているか
   - [ ] 固有名詞・性別が混入していないか
   - [ ] 体言止めのリズムが生きているか
   - [ ] モチーフが通奏低音として機能しているか（複数章ある場合）

   **技術書の確認項目**
   - [ ] 「偏愛エッセイ」のトーンが保たれているか
   - [ ] Why → What → How の順になっているか
   - [ ] キーワードが使われているか
   - [ ] 感嘆・興奮表現が適切に使われているか
   - [ ] 反論先出しパターンが活用されているか

4. 結果をレポートする
   - 項目ごとに ✓ / △ / × で評価
   - △ と × には具体的な箇所を示す
   - 全項目 ✓ なら「スタイルチェック完了：問題なし」

## 注意
- 修正案は出さない（それは /polish の役割）
- どこがDNAと乖離しているかを指摘するだけ

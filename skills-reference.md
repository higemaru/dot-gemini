# スキル一覧 — ~/.gemini/config/skills/

## 初期化

| コマンド | 用途 | 対象 |
|---|---|---|
| `/novel-init` | 小説プロジェクトを初期化する | 小説 |
| `/tech-init` | 技術書プロジェクトを初期化する | 技術書 |

## 執筆ワークフロー

| コマンド | フェーズ | 用途 | 対象 |
|---|---|---|---|
| `/brainstorm` | Plan | アイデア出し。5案を出す | 両方 |
| `/outline` | Plan | 章立てを対話的に作り structure.md を更新する | 技術書 |
| `/draft` | Write | 指定シーン・節の初稿を生成する | 両方 |
| `/polish` | Review | 初稿を推敲し [元の文] → [修正案] 形式で提示する | 両方 |
| `/synopsis` | — | あらすじを生成する（BOOTH掲載・整理・続編着手前） | 小説 |

## 品質チェック

| コマンド | 用途 | タイミング |
|---|---|---|
| `/check-text` | 誤字・脱字・用語統一 | 書いた後、随時 |
| `/check-style` | DNAとの文体乖離チェック | 推敲前 |
| `/check-copyright` | 表現の類似・引用の適切性チェック | 仕上げ前 |

## セッション管理

| コマンド | 用途 |
|---|---|
| `/end` | セッション終了前の作業まとめ・残タスク整理・次回への引き継ぎを作成する |

## チェックスキルの使い分け

```
/check-text      → 誤字・脱字・用語統一（機械的）
/check-style     → DNAとの乖離（質的・文体）
/check-copyright → 著作物に関する確認（類似・引用）
```

## 標準的な1セッションの流れ

```
起動時: SessionStart hook が progress.md / todo.md を表示
  ↓
/brainstorm      アイデアを出す
  ↓
/draft           初稿を書く
  ↓
/check-text      誤字・用語チェック
/check-style     文体チェック
/polish          推敲
  ↓
/end             セッション終了・引き継ぎ作成
```

## スキルの適用範囲とタイミング

| スキル | 対象 | タイミング |
|---|---|---|
| `tech-init` | 全体 | 最初に1回 |
| `outline` | 全体 | 最初に1回 |
| `draft` | 章ごと | 執筆のたびに |
| `polish` | 章ごと | `draft` の直後 |
| `check-text` | 章ごと | `polish` の後 |
| `check-style` | 章ごと（まとめて） | 数章たまったら |
| `check-copyright` | 全体 | 最後に1回 |

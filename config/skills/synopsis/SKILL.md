## 実行手順

1. 用途を聞く（一度だけ）
   - 「あらすじの用途を教えてください」
   - 選択肢の例: BOOTH掲載 / 自分の整理用 / 続編着手前の確認 / その他

2. 用途に応じた長さの目安

   | 用途 | 長さ | スタイル |
   |---|---|---|
   | BOOTH掲載 | 150〜200字 | 購買意欲を引く。ネタバレしない |
   | 自分の整理用 | 400〜600字 | 流れを正確に。ネタバレあり |
   | 続編着手前の確認 | 600〜800字 | 伏線・未回収要素も含める |

3. 素材を読む
   - manuscript/ に原稿があればそちらを優先
   - なければ resources/plot.md を読む

4. あらすじを生成する
   - BOOTH掲載の場合: 作風の雰囲気も伝わるように書く
   - 整理・確認用の場合: 事実ベースで正確に

5. 生成後に確認する
   - 「長さや内容を調整しますか？」

## 注意
- BOOTH掲載用は性別・固有名詞を原稿に合わせる（明示されていなければ書かない）
- 原稿にない情報を補完しない
